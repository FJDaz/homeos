// turbo-all

1. Lire la `ROADMAP.md` via `view_file`.
2. Identifier la mission à exécuter via `grep_search`.
3. Effectuer les `Technical Probes` (lecture de fichiers, status API).
4. Appliquer les modifications via `replace_file_content` ou `write_to_file`.
5. Mettre à jour la `ROADMAP.md` pour passer à l'étape suivante.

> [!NOTE]
> Ce workflow utilise l'annotation `// turbo-all`, ce qui autorise les agents à exécuter les commandes `run_command` considérées comme sûres sans te demander une validation explicite à chaque étape.
