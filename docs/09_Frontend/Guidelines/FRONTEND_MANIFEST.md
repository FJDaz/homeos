# HoméOS — Frontend Manifest (Kimi & Stenciler)

## 🏛️ Philosophie "True HoméOS"
L'interface HoméOS n'est pas un simple outil, c'est un **système de confort cognitif**. Elle repose sur l'immédiateté, la clarté typographique et la fluidité des intentions.

### 1. La Hiérarchie de l'Intuition
- **Typographie Dominante** : Utilisation exclusive de **Univers LT Std**. Toute autre police est proscrite.
- **Grille de 8px** : Fond piqueté de dots gris espacés de 8px pour un alignement millimétré.
- **Micro-interactions** : Chaque action doit avoir un retour visuel (hover, clic, transition).

## 🤖 Les Acteurs du Frontend
### KIMI (Frontend Agent)
- **Rôle** : Architecte visuel et expert en ergonomie.
- **Missions** : Traduire les intentions logiques en structures DOM/CSS performantes.
- **Outils** : TailwindCSS (utilitaires), GSAP (moteur physique), Splitting.js (rythme).

### STENCILER (Moteur de Rendu)
- **Rôle** : Pont entre le Génome et l'affichage.
- **V3 (Layout par Intentions)** : Abandon du positionnement absolu pour un layout élastique basé sur le sens de lecture.

## 🎨 Principes de Design
- **Neutres Chauds** : #f7f6f2 (fond), #9a9a98 (texte secondaire).
- **Vert HoméOS** : #8cc63f (couleur d'accentuation, succès, activation).
- **Radius Stratégique** : 12px pour les panneaux flottants, 0px pour l'IDE Backend (War Room).
- **Glassmorphism** : Utilisation de flous d'arrière-plan pour détacher les couches d'information.
